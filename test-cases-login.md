# Тест-кейсы: страница авторизации

Проект: `https://localhost:8001/user/login`  
Модуль: Авторизация  
Цель: Проверить корректную и безопасную работу формы входа.

## Предусловия

- Приложение доступно по URL: `https://localhost:8001/user/login`
- Есть валидный пользователь:
  - Логин: `Admin`
  - Пароль: `123`
- Есть невалидные данные для негативных сценариев (например, `wrong_user`, `wrong_pass`)

## Формат тест-кейса

- **ID**
- **Название**
- **Шаги**
- **Ожидаемый результат**

---

## Блок A. Базовая доступность и верстка

### TC-LOGIN-001 — Открытие страницы авторизации
- **Шаги:**
  1. Открыть `https://localhost:8001/user/login`
- **Ожидаемый результат:**
  - Страница открылась без ошибок.
  - Видны поле логина, поле пароля, кнопка "Вход".

### TC-LOGIN-002 — Проверка наличия обязательных элементов формы
- **Шаги:**
  1. Открыть страницу логина.
  2. Проверить наличие:
     - поля логина (`#login`)
     - поля пароля (`#password`)
     - кнопки submit (`button[type="submit"]`)
- **Ожидаемый результат:**
  - Все элементы присутствуют и интерактивны.

### TC-LOGIN-003 — Маскирование поля пароля
- **Шаги:**
  1. Открыть страницу логина.
  2. Проверить атрибут поля пароля.
- **Ожидаемый результат:**
  - Поле пароля имеет `type="password"`.
  - Вводимые символы визуально скрыты.

## Блок B. Позитивные сценарии

### TC-LOGIN-010 — Успешная авторизация валидными данными
- **Шаги:**
  1. Ввести логин `Admin`.
  2. Ввести пароль `123`.
  3. Нажать кнопку "Вход".
- **Ожидаемый результат:**
  - Пользователь успешно авторизован.
  - Происходит переход на рабочую страницу.
  - URL соответствует шаблону `*/monitoring/realtime*`.

### TC-LOGIN-011 — Успешная авторизация по Enter
- **Шаги:**
  1. Ввести валидный логин и пароль.
  2. Нажать `Enter` в поле пароля.
- **Ожидаемый результат:**
  - Происходит успешная авторизация, как и при клике на "Вход".

## Блок C. Негативные сценарии

### TC-LOGIN-022 — Неверный логин и пароль
- **Шаги:**
  1. Ввести неверный логин.
  2. Ввести неверный пароль.

## Блок D. Проверки граничных значений и устойчивости

### TC-LOGIN-030 — Логин с пробелами по краям
- **Шаги:**
  1. Ввести логин с пробелами в начале/конце, например `  Admin  `.
  2. Ввести валидный пароль.
  3. Нажать "Вход".
- **Ожидаемый результат:**
  - Система обрабатывает пробелы предсказуемо (trim или корректная ошибка).

### TC-LOGIN-031 — Пароль с пробелами по краям
- **Шаги:**
  1. Ввести валидный логин.
  2. Ввести пароль с пробелами, например ` 123 `.
  3. Нажать "Вход".
- **Ожидаемый результат:**
  - Поведение соответствует политике системы (обычно пароль не тримится).

### TC-LOGIN-032 — Очень длинный логин
- **Шаги:**
  1. Ввести логин длиной > 256 символов.
  2. Ввести пароль.
  3. Нажать "Вход".
- **Ожидаемый результат:**
  - Нет падений UI.
  - Корректная ошибка валидации или отказ в авторизации.

### TC-LOGIN-033 — Очень длинный пароль
- **Шаги:**
  1. Ввести валидный логин.
  2. Ввести пароль длиной > 256 символов.
  3. Нажать "Вход".
- **Ожидаемый результат:**
  - Нет падений UI.
  - Корректная ошибка/отказ.

## Блок E. Безопасность и UX

### TC-LOGIN-040 — Доступ к защищённой странице без авторизации
- **Шаги:**
  1. В новой сессии перейти напрямую на защищённый URL (например, `.../monitoring/realtime`).
- **Ожидаемый результат:**
  - Редирект на страницу логина.

### TC-LOGIN-041 — Ошибка авторизации не раскрывает лишнюю информацию
- **Шаги:**
  1. Выполнить вход с неверными данными.
- **Ожидаемый результат:**
  - Ошибка нейтральная, без уточнения "логин верный/пароль неверный".

### TC-LOGIN-042 — Повторная отправка формы
- **Шаги:**
  1. Ввести валидные данные.
  2. Быстро нажать "Вход" несколько раз.
- **Ожидаемый результат:**
  - Нет дублей запросов/ошибок состояния.
  - Логин выполняется один раз корректно.

### TC-LOGIN-043 — Состояние формы после неуспешного входа
- **Шаги:**
  1. Ввести неверные данные и отправить форму.
- **Ожидаемый результат:**
  - Сообщение об ошибке отображается.
  - Поле логина остаётся/очищается по правилам продукта.
  - Поле пароля обрабатывается безопасно (обычно очищается).

## Блок F. Локализация и выбор языка

### TC-LOGIN-050 — Наличие переключателя языка на странице логина
- **Шаги:**
  1. Открыть страницу авторизации.
  2. Найти элемент переключения языка (кнопка/селект).
- **Ожидаемый результат:**
  - Переключатель языка отображается и доступен для взаимодействия.

### TC-LOGIN-051 — Доступные языки в переключателе
- **Шаги:**
  1. Открыть переключатель языка.
  2. Проверить список доступных языков.
- **Ожидаемый результат:**
  - В списке присутствуют ожидаемые языки (например, `RU`, `EN`).
  - Нет дубликатов и "битых" значений.

### TC-LOGIN-052 — Переключение языка RU -> EN
- **Шаги:**
  1. Установить язык `RU` (если не выбран).
  2. Переключить на `EN`.
- **Ожидаемый результат:**
  - Тексты на странице меняются на английские:
    - заголовок формы,
    - placeholder логина/пароля,
    - текст кнопки входа,
    - сообщения валидации.

### TC-LOGIN-053 — Переключение языка EN -> RU
- **Шаги:**
  1. Установить язык `EN`.
  2. Переключить на `RU`.
- **Ожидаемый результат:**
  - Интерфейс возвращается на русский язык.
  - Все основные тексты локализованы корректно.

### TC-LOGIN-054 — Сохранение выбранного языка после обновления страницы
- **Шаги:**
  1. Выбрать язык (например, `EN`).
  2. Обновить страницу (`F5`).
- **Ожидаемый результат:**
  - После refresh остаётся выбранный язык.
  - Тексты остаются в выбранной локали.

### TC-LOGIN-055 — Сохранение языка после неуспешной авторизации
- **Шаги:**
  1. Выбрать язык `EN`.
  2. Выполнить вход с неверными данными.
- **Ожидаемый результат:**
  - После ошибки авторизации язык не сбрасывается.
  - Сообщение ошибки отображается в выбранной локали.

### TC-LOGIN-056 — Сохранение языка после успешной авторизации
- **Шаги:**
  1. Выбрать язык `EN`.
  2. Выполнить успешный вход.
- **Ожидаемый результат:**
  - На первой открывшейся странице после логина применяется тот же язык.
  - Ключевые элементы интерфейса соответствуют выбранной локали.

### TC-LOGIN-057 — Значение языка в localStorage/cookie (если применимо)
- **Шаги:**
  1. Выбрать язык `EN`.
  2. Проверить наличие/изменение ключа локали в `localStorage` или cookie.
- **Ожидаемый результат:**
  - Значение локали сохраняется ожидаемым способом (storage/cookie).
  - При повторном открытии страницы язык восстанавливается.

### TC-LOGIN-058 — Локализация текста ошибки авторизации
- **Шаги:**
  1. Выбрать язык `EN`.
  2. Ввести неверные креды.
  3. Нажать "Вход".
- **Ожидаемый результат:**
  - Ошибка авторизации отображается на английском.
  - Формулировка корректная и понятная.

### TC-LOGIN-059 — Корректность локализации длинных текстов
- **Шаги:**
  1. Переключаться между языками.
  2. Проверить визуальное отображение длинных строк (кнопки, подсказки, ошибки).
- **Ожидаемый результат:**
  - Тексты не обрезаются критично, не выходят за контейнеры.
  - Верстка остаётся читабельной в каждой локали.
